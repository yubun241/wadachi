藤井工藝（以下「当方」）は、当方が提供するアプリケーション「WADACHI」（以下「本アプリ」）における、利用者の個人情報の取り扱いについて、以下のとおりプライバシーポリシーを定めます。

第1条基本方針

本アプリは、GPS および加速度センサー、ならびに利用者の選択によるBluetooth経由のOBD2車両データ取得機能を用いたタイムアタック計測機能を提供します。本アプリで取り扱うすべての情報は、利用者の端末内でのみ処理され、当方が運営するサーバーへ送信されることは一切ありません。

第2条取得する情報

本アプリは、機能提供のために利用者の端末上で以下の情報を利用します。

1. 位置情報（GPS）

ブラウザの Geolocation API を通じて、計測中に高精度の位置情報を継続的に取得します。これは、利用者が設定したスタート線・フィニッシュ線・セクション線の通過判定、走行軌跡の表示、CSV 記録に使用されます。

2. 加速度センサー情報

ブラウザの DeviceMotion API を通じて、横 G・前後 G の値を取得します。G ボール表示および CSV 記録に使用されます。

3. 端末ローカル保存情報

利用者が作成したコース設定（名称・スタート線座標・セクション座標・目標タイム等）およびベストラップ記録は、ブラウザの localStorage に保存されます。

4. Bluetooth接続情報（OBD2車両データ）

利用者が選択した場合、本アプリはブラウザの Web Bluetooth API を通じて市販の OBD2 Bluetooth アダプターに接続し、エンジン回転数・水温・油温・ブースト圧等の車両データを取得します。この機能は利用者が明示的に接続操作を行った場合にのみ動作し、取得したデータはダッシュボード表示および CSV 記録にのみ使用されます。

重要: 上記いずれの情報も、当方のサーバーへ送信されることはありません。すべての処理は利用者の端末内で完結します。

第3条情報の利用目的

取得した情報は、以下の目的でのみ利用されます。

  • GPS による計測ライン通過判定およびラップタイム算出
  • セクション別スプリットタイムおよびベストラップとの差分表示
  • G フォース・速度の可視化
  • コース設定・走行履歴の保持
  • OBD2接続時の車両情報（回転数・水温・油温・ブースト圧等）のダッシュボード表示
  • 利用者操作による CSV ファイル出力

第4条第三者への提供

当方は、利用者の位置情報・加速度センサー情報・Bluetooth経由で取得した車両データ・コース設定情報を、いかなる第三者にも提供しません。これらの情報は、すべて利用者の端末内でのみ処理されます。

第7条権限の利用について

本アプリは、機能提供のため以下の端末権限の許可をお願いする場合があります。

  • 位置情報: 計測ライン通過判定のために必須です。許可されない場合、計測機能は利用できません。
  • モーションセンサー: G ボール表示のために使用します。許可されない場合、G ボールは動作しませんが、他の機能は利用可能です。
  • Bluetooth: OBD2 アダプターとの接続に使用します。許可されない場合、OBD2 関連機能（車両データ表示）は利用できませんが、他の機能は利用可能です。
  • 画面の常時表示（Wake Lock）: 走行中の画面消灯を防ぐために使用します。

Fujii Kogei ("we", "us") establishes the following Privacy Policy regarding the handling of user information in our application "WADACHI" (the "App").

1.Basic Policy

The App provides time-attack measurement using GPS and accelerometer sensors, as well as optional OBD2 vehicle data retrieval via Bluetooth at the user's discretion. All information handled by the App is processed exclusively on the user's device and is never transmitted to any server operated by us.

2.Information Collected

The App uses the following information on the user's device to provide its functionality.

1. Location Information (GPS)

The App continuously obtains high-accuracy location data through the browser's Geolocation API during measurement. This is used to detect crossings of user-defined start, finish, and section lines, display the driving trace, and produce CSV recordings.

2. Motion Sensor Data

The App obtains lateral and longitudinal G-force values through the browser's DeviceMotion API. This is used for the G-ball display and CSV recording.

3. Locally Stored Information

Course configurations (name, start/section coordinates, target times, etc.) and best lap records created by the user are stored in the browser's localStorage.

4. Bluetooth Connection Data (OBD2 Vehicle Data)

At the user's discretion, the App can connect to a commercially available OBD2 Bluetooth adapter via the browser's Web Bluetooth API to retrieve vehicle data such as engine RPM, coolant temperature, oil temperature, and boost pressure. This feature only operates when the user explicitly initiates a connection, and the retrieved data is used solely for dashboard display and CSV recording.

Important: None of the above information is ever transmitted to our servers. All processing is completed on the user's device.

3.Purpose of Use

The collected information is used solely for the following purposes.

  • GPS-based line-crossing detection and lap-time calculation
  • Display of section split times and deltas against best laps
  • Visualization of G-force and speed
  • Persistence of course settings and lap history
  • Dashboard display of vehicle data (RPM, coolant/oil temperature, boost pressure, etc.) when connected via OBD2
  • CSV file export upon user request

4.Disclosure to Third Parties

We do not provide users' location data, motion sensor data, vehicle data obtained via Bluetooth, or course settings to any third party. All such information is processed exclusively on the user's device.

6.Data Deletion

Users may delete data within the App at any time using the following methods.

  • Individual deletion via the in-App course delete function
  • Bulk deletion via the browser's "Clear site data" function
  • Uninstallation of the App from the device's home screen

7.Device Permissions

The App may request the following device permissions to provide its functionality.

  • Location: Required for line-crossing detection. Without permission, the measurement function cannot be used.
  • Motion Sensors: Used for G-ball display. Without permission, the G-ball will not operate, but other functions remain available.
  • Bluetooth: Used to connect to OBD2 adapters. Without permission, OBD2-related features (vehicle data display) cannot be used, but other functions remain available.
  • Screen Wake Lock: Used to prevent the screen from sleeping during measurement.

8.Security Measures

Since all information handled by the App is processed on the user's device, there is no inherent risk of information leakage over communication channels. However, security management of the device itself (screen lock, biometric authentication, etc.) is the user's responsibility.
